# 공통 코드 리뷰 기준

`/review-code`와 `/review-pr` skill이 공유하는 리뷰 기준이다.

## 리뷰 항목

### a. 코드 품질
- 가독성: 변수/함수 네이밍, 코드 구조
- 중복: 불필요한 반복 코드
- 복잡도: 과도한 중첩, 긴 함수

### b. 버그 가능성
- 엣지 케이스 미처리
- 에러 처리 누락 (unwrap, panic 등)
- 타입 안전성

### c. 보안 이슈
- OWASP Top 10 관점 (인젝션, XSS 등)
- 하드코딩된 시크릿
- 입력 검증 누락

### d. 컨벤션 준수
- CONTRIBUTING.md 커밋 메시지 규칙
- 브랜치 네이밍 규칙
- 코드 스타일 (cargo fmt, eslint)

### e. 테스트
- 변경된 로직에 대한 테스트 존재 여부
- 테스트 커버리지 감소 여부

## 심각도 기준

| 등급 | 의미 | 예시 |
|------|------|------|
| 🔴 Critical | 반드시 수정 필요 | 보안 취약점, 데이터 손실 가능, 패닉 |
| 🟡 Warning | 수정 권장 | 에러 처리 미흡, 성능 저하 가능 |
| 🔵 Suggestion | 선택적 개선 | 네이밍 개선, 리팩토링 제안 |

## 출력 형식

### 요약
- 변경 파일: N개
- 이슈: 🔴 0 / 🟡 0 / 🔵 0

### 상세
파일:라인 형태로 구체적 위치를 명시하고, 문제 설명과 제안을 포함한다.
